## 1. 가상머신(Virtual Machine)의 필요성
* 과거에는 하나의 어플리케이션 프로그램이 특정한 플랫폼에서만 실행 가능한 종속 관계를 가지고 있었다. 예를 들어 i386 ISA는 linux 운영체제를 사용했고, x86은 Window 운영체제를 필요로 했다. 
```
다시 말해, 하드웨어와 운영체제 간 결합이 강했기 때문에 그 관계가 맞지 않으면 프로그램 작동을 원활하게 할 수 없었다.
이러한 문제 의식에서 출발하여 "아무 컴퓨터에서나 내 응용 프로그램을 돌리고 싶다!"라는 목표가 생겼고, 이를 위해 가상머신(Virtual Machine)이 등장하게 되었다.
```
* 즉, 가상 머신은 *Software for Cross-platform Compatibility*를 위해 필요했다.

## 2. 가상머신이란 무엇인가?
* 구체적으로 가상 머신은 컴퓨터 안에 또 다른 컴퓨터를 동작시키는 것이다. 이를 위해 컴퓨터의 주된 부품들인 CPU, RAM, HDD 등의 기능을 소프트웨어적으로 구현해 가상으로 만들어낸다.
* 가상머신에는 **애뮬레이션(Emulation)** 이라는 개념이 존재하는데, 이는 모든 컴퓨터의 부품 기능을 소프트웨어적으로 구현하는 방식이다. 속도는 가장 느리나 범용성은 가장 뛰어나다.
* 또 다른 개념으로는 **가상화(Virtualization)** 이 존재하는데, CPU 등 주요 부품의 기능에서 하드웨어의 기능을 지원받는다. 속도는 빠르나 해당 하드웨어에 종속되므로 범용성이 떨어질 수밖에 없다. 예를들어 CPU를 가상화하면 실제 CPU가 처리하는 기계어 세트에서 크게 벗어날 수 없다.
* **반가상화(Paravirtualization)** 은 기반이 되는 하드웨어, 소프트웨어와 동일하지는 않지만 비슷한 가상 머신에 대한 소프트웨어 인터페이스를 제공하는 기술이다. 완전한 에뮬레이션/가상화를 포기한 방법이라고 이해할 수 있따. 가상 머신에 설치되는 OS에 수정을 가하거나 전용 드라이버를 이용하여 하드웨어에 직접 접근하는 방식을 이용한다. 속도는 가장 빠르지만, 운영체제와 드라이버에 종속되어서 범용성은 가장 떨어진다. Hypervisor를 이용한다.

  * 2. 하나의 컴퓨터 자원을 여러 사용자에게 나누어 주는 상황에서 상호 간섭을 없애고 싶을 때

## 5. 왜 Linux OS를 선호하는가?
* 1. 리눅스는 공개 코드 상에서 보안상 문제가 지적되는 경우가 있지만, 리눅스 커널은 상당히 오랜 기간 동안의 연구, 전세계의 개발자들로부터 지속적인 피드백을 거쳐 완성되었기 때문에 기본적인 완성도가 높고, Open Source Code이기 때문에 프로그램의 오류를 빠르게 확인하여 이에 대한 패치를 제공한다.
* 2. 리눅스 커널이 공개되어 있고, 전세계의 다양한 분야의 개발자들이 참여하여 왔으며 하드웨어 관련 기업들의 참여가 확산되어 있다. 이에 따라 리눅스 커널은 폭넓은 하드웨어 장치들을 지원하는 드라이버들을 포함하고 있다.
* 3. 리눅스는 거의 대부분의 C언어와 어셈블리 언어로 작성되어 있다. C언어 자체가 특정 기계에 비의존적이기 때문에 프로그램을 다른 기계의 시스템으로 Porting하는 것이 용이하다. 이런 이유로 OS자체도 하드웨어에 적합하게 변형이 가능하다.
* 4. UNIX 시스템과 리눅스 시스템은 거의 모든 프로그래밍 언어를 제공한다. 특히 GNU 소프트웨어가 무료로 제공되면서 수백, 수천만원의 프로그래밍 툴을 사용하지 않아도 된다. 또한 모든 시스템의 소스가 개방되어 있기 때문에 소프트웨어 개발에 개방적이다.

## 6. Linux OS의 특징
* 1. 리눅스 운영체제는 모놀리틱(Monolithic) 커널을 사용하는데, 이는 프로세스, 메모리, 파일 시스템, I/O 관리, 네트워크의 모든 커널의 기능을 커널의 내부에 System call과 Interrupt 처리 부분으로 포함하여 제공하는 것이다. 따라서, System Call에 의한 서비스가 빨라진다. 
* 2. 또한, 모놀리틱 커널이지만 어느정도 운영체제가 동작하는 중에도 커널 코드를 추가하거나 삭제할 수 있다. 리눅스 커널은 이러한 기능을 **모듈(Module)**이라는 방식으로 구현한다.
* 3. 이외에도 커널 스레드와 멀티스레드를 지원한다는 장점이 존재한다.
